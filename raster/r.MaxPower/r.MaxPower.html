NAME

r.MaxPower - RaPlaT - MaxPower module (v24mar2021)

KEYWORDS

raster, MaxPower

SYNOPSIS

r.MaxPower
r.MaxPower --help
r.MaxPower [-q] cell_input=string [generate=string] [rx_threshold=float] [chan_type=string] [bandwidth=float] output=name [table=string] [driver=string] [database=string] [cell_num=integer] [dbperf=integer] [--overwrite] [--help] [--verbose] [--quiet] [--ui]

Flags:

-q
Quiet
--overwrite
Allow output files to overwrite existing files
--help
Print usage summary
--verbose
Verbose module output
--quiet
Quiet module output
--ui
Force launching GUI dialog

Parameters:

cell_input=string [required]
Cells data table
generate=string
Selection of the generated output contents
Options: rss-max, coverage, rss-sum, rss-maxix, lte-rssi, lte-rsrp, lte-rsrq, lte-cinr, lte-maxspecteff, lte-maxthrput, lte-interfere
Default: rss-max
rx_threshold=float
Minimum received power [dBm] for radio signal coverage
Default: -999
chan_type=string
Channel type - Gaussian or Rayleigh (currently only Gaussian)
Options: gaussian
Default: gaussian
bandwidth=float
Bandwidth [MHz] (required for LTE computations)
Default: 5
output=name [required]
Name for output raster map
table=string
Table name
driver=string
Driver name
Options: dbf, mysql, odbc, ogr, pg, sqlite, none, csv
Default: none
database=string
Database name
Default: $GISDBASE/$LOCATION_NAME/$MAPSET/dbf/
cell_num=integer
Number of successive path loss values to be written in the table
Default: 5
dbperf=integer
Database insert performance (rows/INSERT; 99: special fast mode via CSV)
Options: 1-99
Default: 20
